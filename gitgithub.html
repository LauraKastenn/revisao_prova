GIT:

O Git é uma ferramenta padrão para desenvolvimento de software. Git é um VCS, que significa "Version Control System".
Utilizado para gerenciar o código-fonte de projetos, permitindo que múltiplas pessoas trabalhem simultaneamente sem conflitos.

Comandos no git:

1. Configuração Inicial
- git config --global user.name "Seu Nome": Define o nome do usuário que será associado aos commits.
- git config --global user.email "seuemail@exemplo.com": Define o e-mail do usuário que será associado aos commits.

2. Criação e Clonagem de Repositórios
- git init nome-do-repositorio: Cria um novo repositório Git vazio no diretório especificado.
- git clone url-do-repositorio: Clona um repositório existente a partir de uma URL, criando uma cópia local.

3. Gerenciamento de Alterações
- git add nome-do-arquivo: Adiciona um arquivo específico à área de staging, preparando-o para o commit.
- git add: Adiciona todos os arquivos modificados na pasta atual à área de staging.
- git commit -m "Mensagem do commit": Registra as alterações na área de staging com uma mensagem descritiva.

4. Visualização de Alterações
- git status: Mostra o estado atual do repositório, incluindo arquivos modificados, em staging e não rastreados.
- git log: Exibe o histórico de commits, mostrando as mensagens de commit, autores e datas.

5. Branching e Merging
- git branch nome-da-branch: Cria uma nova branch (ramificação) no repositório.
- git checkout nome-da-branch: Muda para a branch especificada, permitindo trabalhar nela.
- git merge nome-da-branch: Mescla as alterações da branch especificada na branch atual.

6. Sincronização com Repositórios Remotos
- git push origin nome-da-branch: Envia as alterações locais da branch especificada para o repositório remoto.
- git pull origin nome-da-branch: Baixa e integra as alterações da branch especificada do repositório remoto.

7. Outros Comandos Úteis
- git checkout -- nome-do-arquivo: Reverte as alterações em um arquivo específico para a última versão confirmada.
- git reset nome-do-arquivo: Remove um arquivo da área de staging, mas mantém as alterações no diretório de trabalho.

GITHUB:

O GitHub é uma plataforma baseada em nuvem em que é possísvel armazenar, compartilhar e trabalhar com outras pessoas para escrever códigos.

Comandos no github

Criar um novo repositório: Clique em "New" no GitHub para iniciar um repositório.

Clonar um repositório remoto: git clone https://github.com/usuario/nome-do-repositorio.git - Cria uma cópia local do repositório remoto.

Enviar alterações: git push origin nome-da-branch - Envia commits locais para o repositório remoto.
Baixar alterações: git pull origin nome-da-branch - Atualiza a branch local com mudanças do repositório remoto.

Listar branches remotas: git branch -r - Mostra todas as branches disponíveis no repositório remoto.
Criar nova branch remota: git push origin nome-da-branch - Cria uma nova branch no repositório remoto.

Criar um Pull Request: Acesse "Pull requests" e clique em "New pull request" para propor alterações.

Criar uma nova Issue: Vá até "Issues" e clique em "New issue" para relatar um problema ou sugestão.

Ver atividades do repositório: Acesse "Insights" para visualizar estatísticas e gráficos do repositório.
